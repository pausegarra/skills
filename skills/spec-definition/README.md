# spec-definition

`spec-definition` is a skill for turning user stories into clear, reviewable product specifications.

## What It Does

- Applies when users ask to define, draft, complete, or refine a product specification from a user story.
- Creates a preliminary specification with relevant product, UX, workflow, validation, and functional details.
- Lists non-technical assumptions so the user can explicitly accept or reject them.
- Guides rejected assumptions one at a time with progress tracking and concrete replacement options.

## Assumption Handling

- Focuses on product, business, UX, content, workflow, permission, validation, notification, and functional behavior decisions.
- Excludes purely technical assumptions such as frameworks, databases, architecture, file names, or implementation details.
- Provides four concrete alternatives plus `Otra` for each rejected assumption.
- Produces the final specification only after the user asks for it.

## Files

- `SKILL.md`: Full operational instructions, response formats, and assumption-handling rules.
